# Checkpoint: 2026-03-16

## Session Summary

### Architecture Changes
- **Removed Python dependency entirely**: deleted `asr_service/`, `.venv/`, `requirements.txt`, and all Python references. The app is now a pure Swift/C project.
- **Direct sherpa-onnx C API integration**: rewrote `ASRService.swift` to call sherpa-onnx via a bridging header and dylib linking, eliminating the previous Python IPC layer.
- **Punctuation restoration**: added ct-transformer model support through sherpa-onnx for automatic punctuation.

### Xcode Project Updates
- Updated `project.pbxproj` with bridging header, library search paths, linker flags, and a Copy Files build phase for dylibs.
- Production builds only search Bundle Resources and Application Support for models (no dev path fallbacks using `#filePath`).

### New Components
- **ModelDownloader.swift**: chunked parallel download with resume support (inspired by OwlUploader's R2Service). Downloads ASR (~838MB), punctuation (~270MB), and VAD (~2MB) models to `~/Library/Application Support/OwlWhisper/models/`.
- **FloatingIndicator.swift**: waveform animation during recording, blue dot animation during transcription, fixed at screen bottom center (AX API unreliable for cursor position across apps).

### UI Changes
- **Unified settings window**: deleted `OnboardingWindow`, merged onboarding guide and settings into a single `SettingsWindowController`.
- **Model auto-download** integrated into settings window.
- **Menubar transcribing animation**: changed from spinner to three-dot bounce.
- **Version + author info** added at bottom of settings window.

### Bug Fixes
- Fixed keyUp detection in `HotkeyManager` (don't check modifier flags on keyUp).
- Added `CGEventTap` retry timer (every 5s) when Input Monitoring permission not yet granted.
- Fixed model path resolution for production (removed `#filePath` and project directory fallbacks).

### Decisions
- Input Monitoring permission auto-granted for signed apps from `/Applications`; removed from settings UI.
- GitHub CDN doesn't return `Content-Length`; using hardcoded file sizes as fallback for progress display.
- Single settings window replaces separate onboarding flow (user preference for simplicity).

### Testing
- Full production test passed: clean install from `/Applications`, model download, permissions grant, recording, transcription with punctuation.

### Pending

- **Internationalization (i18n)**: Chinese + English, default English -- not started.
- **Version update management**: planning to use Sparkle framework -- not started.
