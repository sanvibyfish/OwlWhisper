# REQ-001: 本地语音输入工具

> 状态：🔵 Draft | 优先级：P1 | 预估工时：1周 | 创建日期：2026-03-16

---

## 1. 需求概述

基于 sherpa-onnx + FireRedASR2 构建纯本地离线的中文语音输入工具，实现类 macOS 听写体验：全局快捷键触发，任意 App 内可用。

---

## 2. 背景与动机

- macOS 自带听写依赖 Apple 服务，中文准确度一般，且有隐私顾虑
- 当前中文离线 ASR 模型（FireRedASR2）已达到工业级准确度（2.89% CER），完全可以替代云服务
- 日常工作中大量中文文字输入场景（内容写作、笔记、消息），语音输入可显著提效
- sherpa-onnx 提供完整的离线推理框架，部署成本极低

---

## 3. 用户故事

作为日常在 Mac 上工作的用户，我希望按住快捷键说中文，松开后文字自动出现在光标位置，全程离线不联网，准确度不低于主流云服务。

---

## 4. 功能描述

### 4.1 核心功能

| 功能 | 描述 |
|------|------|
| 按住说话 | 按住全局快捷键开始录音，松开停止 |
| 自动转写 | 录音结束后通过 VAD 切段 + FireRedASR2 转写为文字 |
| 自动粘贴 | 转写结果写入剪贴板，模拟 Cmd+V 粘贴到当前光标位置 |
| 全局可用 | 在任意 App 的任意输入框中均可使用 |
| 纯本地运行 | 不依赖网络，所有处理在本机完成 |
| 状态指示 | menubar 图标反映当前状态（就绪/录音/转写） |

### 4.2 用户流程

```
用户按住快捷键
  → 播放提示音 / menubar 图标变化（录音中）
  → 麦克风开始采集音频
用户松开快捷键
  → 停止录音
  → VAD 切段（去除首尾静音）
  → FireRedASR2 转写
  → 文字写入剪贴板
  → 模拟 Cmd+V 粘贴
  → 恢复 menubar 图标（就绪状态）
```

### 4.3 边界情况

| 场景 | 处理方式 |
|------|----------|
| 按住但不说话 | VAD 检测无语音，不执行转写，不粘贴 |
| 录音时间过长（>60s） | 自动停止录音，转写已录内容 |
| 转写结果为空 | 不执行粘贴操作 |
| 当前无活跃输入框 | 仍写入剪贴板，粘贴可能无效果但不报错 |
| 麦克风权限未授予 | 首次启动时提示授权 |

## 5. 技术方案

### 5.1 架构设计

```
┌──────────────────────────────────────────┐
│              voice-input                  │
│                                           │
│  ┌─────────┐  ┌──────────┐  ┌──────────┐ │
│  │ Hotkey   │→│ Recorder │→│ ASR      │ │
│  │ Listener │  │ (mic)    │  │ Engine   │ │
│  └─────────┘  └──────────┘  └──────────┘ │
│       ↑                          ↓        │
│  ┌─────────┐               ┌──────────┐  │
│  │ Menubar │               │ Output   │  │
│  │ Status  │               │ (paste)  │  │
│  └─────────┘               └──────────┘  │
└──────────────────────────────────────────┘

ASR Engine 内部:
  音频 → silero-vad (切段) → FireRedASR2 (转写) → 文字
```

### 5.2 API 设计

无外部 API，纯本地进程。组件间通过 Swift 函数调用，ASR 通过 sherpa-onnx C API 直接调用。

### 5.3 数据模型

无持久化数据。音频在内存中处理，转写结果写入系统剪贴板后丢弃。

### 5.4 第三方依赖

| 依赖 | 版本 | 用途 |
|------|------|------|
| sherpa-onnx (C API) | latest | ASR 引擎 + VAD，通过桥接头直接在 Swift 中调用 |
| onnxruntime | 1.23.2 | sherpa-onnx 的推理后端 |

**ASR 模型：**
- 名称：`sherpa-onnx-fire-red-asr2-zh_en-int8-2026-02-26`
- 语言：中文（普通话 + 20+ 方言）+ 英文
- 模式：非 streaming（离线转写），通过 VAD 切段实现准实时
- 量化：int8（体积小，速度快）

| AC-6 | 内存占用 ≤ 500MB（含模型） | P1 |
